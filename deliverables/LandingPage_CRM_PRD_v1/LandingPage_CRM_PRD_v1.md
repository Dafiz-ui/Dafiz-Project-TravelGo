# PRD v1 — Landing Page CRM (Customer Relationship Management)

## 1) Ringkasan
Landing page CRM bertujuan mengubah pengunjung menjadi **lead** (email) melalui pesan yang jelas: CRM membantu tim penjualan mengelola prospek, follow-up, dan pipeline dengan rapi.

## 2) Tujuan Produk
- Memberi pemahaman cepat: CRM ini untuk siapa dan manfaat utamanya.
- Mengarahkan tindakan: pengguna meninggalkan email untuk menerima demo / onboarding.

## 3) Persona Target
- **Sales/Account Executive** yang mengelola banyak prospek manual (spreadsheet/WhatsApp).
- **Admin/Manager** yang butuh visibilitas pipeline dan aktivitas follow-up.

## 4) Problem yang Diselesaikan
- Data prospek tersebar dan mudah hilang.
- Follow-up tidak konsisten sehingga lead menurun.
- Kurang visibilitas status deal (pipeline tidak transparan).

## 5) Solusi (Positioning)
CRM menyatukan prospek, riwayat komunikasi, dan status pipeline sehingga tim dapat bekerja konsisten.

## 6) Value Proposition (pesan inti)
- Kelola prospek dalam satu tempat.
- Jadwal follow-up otomatis/terstruktur.
- Lihat pipeline real-time untuk keputusan cepat.

## 7) Landing Page Goal (AIDA)
- **Attention**: hero headline spesifik tentang “kelola prospek & pipeline”.
- **Interest**: ringkasan problem→solusi.
- **Desire**: fitur utama (kartu fitur) dan testimoni singkat.
- **Action**: form email sederhana + CTA.

## 8) Struktur Halaman (Top / Middle / Bottom)

### TOP — Navbar & Hero
- Navbar: logo + 2 link (Fitur, FAQ) + tombol “Mulai Gratis”.
- Hero:
  - Pre-title: konteks CRM
  - Headline: manfaat utama 1 kalimat
  - Subheadline: 1-2 kalimat bukti nilai
  - Primary CTA: “Mulai Gratis” (scroll ke form)
  - Secondary CTA: “Lihat Demo” (anchor ke section demo/FAQ)

### MIDDLE — Problem → Solution + Features + Testimoni + FAQ
- Section Problem/Solution: 3 poin problem, 3 poin solusi.
- Feature Section: minimal 6 card (1 ide per card).
- Testimoni: 3 kutipan singkat.
- FAQ: 4 pertanyaan paling umum.

### BOTTOM — Form CTA + Footer
- CTA ulang: 1 kalimat pengulangan nilai + tombol “Mulai Gratis”.
- Form email:
  - input email
  - tombol submit
  - microcopy: “Tidak spam. Kami kirim contoh demo.”
- Footer:
  - link informasi (Fitur, FAQ)
  - copyright

## 9) Success Metrics (metrik)
- Conversion rate visitor → submit email.
- Scroll depth ke section form.

## 10) Evaluasi PRD v1 (Checklist)
- [x] Persona & problem jelas
- [x] Value prop ringkas
- [x] AIDA dan struktur Top/Middle/Bottom didefinisikan
- [ ] CTA/Copy perlu disempurnakan (akan di PRD v2)
- [ ] UX friction form (validasi, feedback) perlu ditambah (PRD v2)

## 11) Hasil yang Akan Dibuat
- Konten landing page versi awal mengikuti struktur di atas.
- Dokumen ini menjadi acuan copy & section layout untuk implementasi.
